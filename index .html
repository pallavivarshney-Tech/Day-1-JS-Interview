Question 3

What are the differences between var, let and const?

let- we can declare the variable name in one line or in the same line itself. Two variables with same names aren't possible in let. we can reassign/update the value if already asssigned. Let is a block-scoped. Let doesn't supports hoisting.

Hoisting - it brings the variables and functions at the top of thecode it supports only variable declaration but not its initialisation.

Const- we have to declare and initialise the variable name in same line. Two variables with same names aren't possible in const. we can't reassign/update the value if already initialised. Let is a block-scoped. Let doesn't supports hoisting.

Var- we can declare the variable name in one line or in the same line itself. Two variables with same names are possible in var. we can reassign/update the value if already asssigned. Let is a global-scoped. Var supports hoisting.

Question 4

What is execution context?

The Execution Context contains the code that's currently running, and everything that aids in its execution. During the Execution Context run-time, the specific code gets parsed by a parser, the variables and functions are stored in memory, executable byte-code gets generated, and the code gets executed.

There are two kinds of Execution Context in JavaScript:
Global Execution Context (GEC)
Function Execution Context (FEC)

Global Execution Context:-Whenever the JavaScript engine receives a script file, it first creates a default Execution Context known as the Global Execution Context (GEC). The GEC is the base/default Execution Context where all JavaScript code that is not inside of a function gets executed.

Function Execution Context:-Whenever a function is called, the JavaScript engine creates a different type of Execution Context known as a Function Execution Context (FEC) within the GEC to evaluate and execute the code within that function. Since every function call gets its own FEC, there can be more than one FEC in the run-time of a script.
